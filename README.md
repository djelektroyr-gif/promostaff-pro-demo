# PROMOSTAFF PRO — демо-бот (облегчённый сценарий)

Отдельный маленький репозиторий: один Telegram-бот на **aiogram 3** + **SQLite** (`promostaff_demo.db` создаётся рядом с кодом при первом запуске).

## Сценарий демо

1. **Исполнитель** и **заказчик** регистрируются через `/start`.
2. **Админ** (ваш `ADMIN_USER_ID`) — команда `/admin`: список заказчиков (с удалением тестовых), создание **проекта с выбором заказчика из списка**, создание **смены**, назначение исполнителя.
3. Заказчик: «Мои смены», детали смены, **чат смены**, **задачи**, отчёт с часами и суммами.
4. Исполнитель: подтверждение выхода → **чек-ин** (гео + фото) → **чек-аут**, «Мои задачи».

Напоминания за N часов до смены в коде **не реализованы** (в полном PRO — планировщик); при необходимости добавьте `aiogram` + фоновый цикл или APScheduler.

## Запуск

```bash
cd promostaff-pro-demo
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt
copy .env.example .env   # заполните BOT_TOKEN и ADMIN_USER_ID
python main.py
```

`ADMIN_USER_ID` должен быть **> 0** (см. `config.py`).

## Полевой тест «две смены»

1. Зарегистрируйте заказчика (отдельный аккаунт или второй телеграм).
2. С аккаунта админа: `/admin` → создать проект → выбрать этого заказчика → имя проекта.
3. Создать две смены (две строки с датами).
4. Назначить исполнителя на каждую смену.
5. Зайти исполнителем: подтвердить, чек-ин/аут по очереди.

Удаление тестового заказчика: `/admin` → «Заказчики» → «Удалить …» (удаляются его проекты, смены, назначения, задачи, сообщения чата).
