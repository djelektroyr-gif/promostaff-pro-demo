# PROMOSTAFF PRO — демо-бот (облегчённый сценарий)

Отдельный репозиторий: один Telegram-бот на **aiogram 3** + **SQLite**.  
База `promostaff_demo.db` создаётся рядом с кодом при первом запуске.

## Что умеет демо

### Исполнитель
- регистрация через `/start` (ФИО, телефон, профессия);
- просмотр своих смен;
- подтверждение выхода;
- чек-ин (гео + фото);
- чек-аут (гео + фото или пропуск);
- просмотр и выполнение задач;
- чат смены.

### Заказчик
- регистрация через `/start` (компания, контакт, телефон);
- просмотр своих проектов и смен;
- карточка смены с назначенными исполнителями;
- постановка задач исполнителям;
- чат смены;
- отчёт по смене (часы/выплаты).

### Админ (`ADMIN_USER_ID`)
Команда `/admin` открывает панель:
- **Метрики**: исполнители, заказчики, проекты, открытые смены;
- **Исполнители**: список, удаление, статусы заявок (`new`, `reviewed`, `approved`, `rejected`);
- **Заказчики**: список и каскадное удаление;
- **Создать проект**;
- **Создать смену**;
- при создании смены можно задать координаты площадки и радиус допуска для чек-ин/чек-аут;
- **Управление сменами**: закрыть / удалить (каскадно);
- **Назначить на смену** (в назначение попадают `approved` исполнители);
- **Генератор тест-данных**;
- **Лог действий админа**.

## Команды

- `/start` — старт и выбор роли/меню;
- `/admin` — админ-панель (только `ADMIN_USER_ID`).

Остальные действия выполняются кнопками в интерфейсе.

## Сценарий для тестовой группы

Если добавить бота в рабочий тестовый чат проекта:

- бот отправит приветствие;
- покажет 2 кнопки с deep-link:
  - `👷 Я исполнитель` → `.../start=register_worker`
  - `🏢 Я заказчик` → `.../start=register_client`

После нажатия пользователь переходит в ЛС бота и **сразу** попадает в нужный сценарий регистрации (без лишних шагов).

## Запуск

```bash
cd promostaff-pro-demo
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt
copy .env.example .env   # заполните BOT_TOKEN и ADMIN_USER_ID
python main.py
```

`ADMIN_USER_ID` должен быть **> 0** (см. `config.py`).

## Быстрый e2e-прогон

1. С аккаунта админа: `/admin` → **Генератор тест-данных**.
2. Проверить `/admin` → **Метрики** (данные появились).
3. Открыть `/admin` → **Статусы исполнителей** и выставить часть в `approved`.
4. Открыть `/admin` → **Назначить на смену** и назначить исполнителей.
5. С аккаунта исполнителя: открыть «Мои смены» → подтвердить → чек-ин → чек-аут.
6. С аккаунта заказчика: открыть смену → поставить задачу → проверить отчёт.
7. Проверить `/admin` → **Лог действий** (фиксируются ключевые действия).

## Важные заметки

- Напоминания за N часов до смены в демо **не реализованы** (в полном PRO это отдельный планировщик).
- При удалении смены удаляются связанные назначения/задачи/сообщения чата.
- При удалении исполнителя его назначения удаляются, а задачи отвязываются (assigned_to = NULL).
